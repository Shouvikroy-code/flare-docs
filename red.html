Redesign of Data Store Functionality to Improve Usability and Experience

Previously, users had to upload the same set of files multiple times for each configuration, leading to inefficiencies and redundancy. The "Configuration Name" dropdown on the Data Store UI required users to link each set of data store files to a specific configuration, even when the files were common across configurations. The Data Store name field was auto-populated and non-editable, preventing users from defining meaningful names for stored datasets.

This enhancement improves the usability of the Data Store by enabling users to upload a single set of files and reuse them across configurations, eliminating the need for repeated uploads. The enhancement also introduces UI and functional improvements, making the Data Store more intuitive and user-friendly.

New Functionality

  • Data Store Enhancements:

    • Users can now manually define a Data Store name when uploading files, making it easier to organize and retrieve data. For example, you can now save a set of files based on the month they’re uploaded.

    • If a Data Store name already exists, uploading new files will replace the existing files.

    • Validation ensures that Data Store names contain only letters and digits (no spaces or special characters).

    • The "Practitioner" box is now renamed to "Professional."

    • The "Configuration Name" dropdown has been removed from the Data Store UI.

  • Manual Run Screen Updates:

    • The “Load Data Combination from” label has been renamed to “Data Store”, and its info text now explains that users can select a set of uploaded files as inputs.

    • The dropdown menu allows users to select from available Data Store names created on the Data Store screen.

    • If any required claim input files (Professional, Inpatient or Outpatient) are missing, a message prompts users to enter From Date, To Date, and Cutoff before proceeding.

    • When all required claim files are available in the selected Data Store, From Date, To Date, and Cutoff fields are disabled, and a message confirms the presence of Professional, Inpatient, and Outpatient claims.

    • Users can now execute the Run process without entering From Date, To Date, or Cutoff when a Data Store is selected.

    • Users cannot execute the Run process without entering From Date, To Date, or Cutoff when a Data Store is not selected.

    • The Date of Service field remains available at the bottom of the form since this does not depend on the input claims data.

Note The Data Store retains files for 90 days, after which they are removed from the UI.

For more information, see "Data Store" and "Manual Run Tab" section of "Generating a In Network MRFs" in HR MRF Suite user guide.
